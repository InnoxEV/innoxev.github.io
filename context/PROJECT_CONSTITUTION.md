

# INNOX WEBSITE CONSTITUTION (v4.0)

## 1. Mission
Build a leading EV platform for Arabic-speaking users focused on BYD, tools, apps, and real-world solutions across MENA.

---

## 2. Product Vision
Innox is not just a website.
It is a structured platform combining:
- Content
- Tools
- Services

Goal: Help EV users solve real problems.

---

## 3. Core Pillars

- Education  
  Guides, fixes, tutorials  

- Utility  
  Apps, tools, real solutions  

- Monetization  
  Affiliate, services, products  

---

## 4. System Architecture

- index.html → Welcome Page (language selection)
- /en/index.html → English Homepage (main structure)
- /ar/index.html → Arabic Homepage (RTL version)

- /specs → Feature definitions (HOW to build)
- /context → Product vision & strategy (WHY we build)
- PROJECT_CONTEXT.md → Rules for AI (Codex)

---

## 5. Product Roadmap

### Phase 1 (Current - MVP)
- Homepage (EN + AR)
- Apps Page
- Accessories Page

### Phase 2 (Growth)
- BYD Issues & Fixes
- SEO content (guides & tutorials)

### Phase 3 (Monetization)
- Affiliate links
- Featured listings

### Phase 4 (Advanced)
- EV Support System
- Direct product selling (EV chargers)

---

## 6. Development System

- Every feature MUST have a spec
- Spec includes:
  - Goal
  - Structure
  - UI behavior
  - Acceptance criteria

- EN spec = source of truth
- AR version = adaptation layer (NOT separate spec)

---

## 7. Localization Strategy

- English = base version
- Arabic = same structure + translation + RTL

Rules:
- Do NOT change structure
- Do NOT redesign layout
- Only adapt language and direction

---

## 8. Workflow

1. Select feature  
2. Write spec  
3. Build with Codex  
4. Review UI  
5. Commit & push  

---

## 9. Rules

- No feature without clear value  
- Do not skip phases  
- Keep UI minimal (Apple-style)  
- Focus on execution  
- Limit "Coming Soon" items  

---

## 10. Current Focus

- Apps Page (priority)
- First App Details Page
- Arabic UI refinement

---

## 11. Mindset

This is not a website.
This is a product system.

Focus on:
- Structure
- Clarity
- Execution
