# Arabic Homepage Rules - Innox

## Target File
/ar/index.html

## Base Rule
The Arabic homepage must follow /specs/homepage.md exactly in:
- Structure
- Sections
- Components
- Layout
- Keep visual consistency with the English version

Do NOT change or add sections.

---

## Language Adaptation

- Translate all text to Arabic
- Keep meaning, not literal translation
- Maintain short, clear phrases

---

## RTL Layout

- Use dir="rtl"
- Text aligned right
- Layout must feel natural in Arabic reading direction
- Ensure components (cards, buttons) follow RTL flow naturally

---

## Typography

- Use Tajawal font for Arabic
- Increase line-height for readability (1.7–1.8)
- Avoid tight spacing

---

## UI Behavior

- Same interactions as English
- Same hover effects
- Same spacing system

---

## Navigation

- Keep same structure:
  - /ar/index.html
  - /ar/apps.html
  - /ar/pages/accessories.html

---

## Constraints

- Do NOT redesign layout
- Do NOT change structure
- Only adapt for Arabic language and RTL

---

## Implementation Scope
- Update /ar/index.html only
- Use existing CSS (style.css)
- Do not create new files
